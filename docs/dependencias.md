En el contexto de la ingeniería de software, una dependencia es un paquete de código externo, módulo, biblioteca o framework preexistente que un proyecto u aplicación requiere para compilarse, ejecutarse o realizar funciones específicas. Pueden ser externas (mantenidas por terceros) o internas (desarrolladas y mantenidas dentro de la organización).
En lugar de programar todas las funcionalidades desde cero (como el manejo de peticiones HTTP, algoritmos criptográficos o conexión a bases de datos), los desarrolladores integran estas piezas de software empaquetadas. Esto fomenta la reutilización de código y la modularidad.
Las dependencias se gestionan a través de gestores de paquetes (como pip o poetry en Python), los cuales leen archivos de configuración (pyproject.toml) para descargar e instalar las versiones correctas de esos paquetes y de sus respectivas sub-dependencias (lo que se conoce como árbol de dependencias transitivas). Las dependencias incluidas en nuestro proyecto son: 

1. fastapi==0.104.0
•	Definición: Es un framework web moderno del tipo backend (su fuerte) y de alto rendimiento diseñado para construir APIs REST, microservicios, backends poara aplicaciones web y móviles y APIs para modelos de ML e IA.
•	Arquitectura y Rol: FastAPI no maneja directamente la red, sino que es un framework a nivel de aplicación basado en el estándar ASGI (Interfaz de puerta de enlace de servidor asíncrono, que es intermediaria de servicios de protocolo de red y apps de Python), lo que permite manejar tareas de forma asíncrona (async/await) de manera nativa. Su núcleo está fuertemente acoplado con los Type Hints (anotaciones/sugerencias de tipo) estándar de Python; para validar automáticamente los datos de entrada y salida, reduciendo errores comunes hasta en un 40%..
•	Mecanismo: Utiliza estas anotaciones para realizar dos tareas fundamentales: el enrutamiento de peticiones (mapear URLs a funciones Python) y la validación automática de datos de entrada/salida y serialización (apoyándose internamente en Pydantic). Además, auto-genera documentación técnica interactiva (Swagger UI y ReDoc) basándose en el estándar OpenAPI.

2. uvicorn[standard]==0.24.0
•	Definición: Es una implementación de servidor web ASGI (Interfaz de puerta de enlace de servidor asíncrono), que utiliza las características de programación asíncrona en Python para mejorar el rendimiento de app web en Python.
•	Arquitectura y Rol: Mientras FastAPI define qué hacer con una petición HTTP, WebSockets, entre otras; uvicorn es el encargado de recibir esa petición HTTP desde la red, traducirla al formato ASGI y pasarla a FastAPI de manera asíncrona.
•	El sufijo [standard]: Al incluir [standard], se indica al gestor de paquetes que instale uvicorn junto con sus dependencias opcionales recomendadas para optimizar el rendimiento. Esto típicamente incluye uvloop (un reemplazo del bucle de eventos en la biblioteca estándar asyncio de Python) y httptools (un parser, es decir, analizador HTTP de Node.js ultrarrápido).

3. pydantic>=2.0.0,<3.0.0 
•	Definición: Es la biblioteca de validación de datos y gestión de esquemas más utilizada en el ecosistema moderno de Python. En lugar de escribir docenas de si isinstance() y funciones de validación personalizadas, define la estructura de datos una sola vez utilizando la conocida sintaxis de Python. Se encarga de validar los datos entrantes, convertir los tipos cuando es necesario y proporcionarmensajes de error claros cuando falla la validación.
•	Arquitectura y Rol: Pydantic garantiza que la estructura y los tipos de datos que entran o salen del sistema sean correctos en tiempo de ejecución. Funciona definiendo modelos (clases) donde los atributos tienen Type Hints. Si un dato recibido (por ejemplo, un JSON en una petición POST) no coincide con el tipo esperado, Pydantic lanza un error de validación detallado, evitando que datos corruptos avancen hacia la lógica de negocio o la base de datos del extractor.
•	Nota de versión: La versión 2 de Pydantic fue reescrita en Rust (a través de pydantic-core), lo que incrementó drásticamente su rendimiento de validación en comparación con la versión 1.x.

4. pydantic-settings>=2.0.0 
•	Definición: Es un paquete derivado del núcleo de Pydantic, diseñado específicamente para la gestión de configuraciones, variables de entorno, archivos JSON y más.
•	Arquitectura y Rol: En arquitecturas de clean code y buenas prácticas (como los Twelve-Factor Apps), las configuraciones (credenciales de bases de datos, claves API, modos de ejecución) deben estar estrictamente separadas del código fuente. Este paquete te permite crear un modelo (clase) BaseSettings. Al instanciarlo, pydantic-settings lee automáticamente las variables de entorno del sistema operativo o de un archivo .env, las parsea, las valida y las convierte a los tipos de datos correctos en Python (por ejemplo, convierte el string "True" de una variable de entorno a un booleano True).

5. python-multipart==0.0.6
•	Definición: Es un parser (analizador sintáctico) asíncrono para flujos de datos multipart/form-data.
•	Arquitectura y Rol: En el protocolo HTTP, cuando un cliente envía datos de un formulario complejo o sube archivos binarios (como un archivo .pdf), la petición se codifica en un formato llamado multipart/form-data. FastAPI no incluye un parser para esto por defecto. Requiere explícitamente python-multipart para poder descodificar el body de la petición y extraer los archivos y campos del formulario. Sin esta dependencia, los endpoints de tu pdf-extractor que utilicen las clases File o UploadFile de FastAPI para recibir los PDFs simplemente arrojarán un error y no podrán funcionar.
