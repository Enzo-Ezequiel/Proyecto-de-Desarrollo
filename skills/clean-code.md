# Clean Code

##  Filosofía

El código limpio es aquel que puede ser leído y mantenido por otros desarrolladores.

---

##  Cuándo aplicarlo

- Desarrollo de nuevas funcionalidades
- Refactorización
- Revisiones de código

---

##  Nombres

- Usar nombres descriptivos
- Clases → sustantivos
- Métodos → verbos

---

##  Funciones

- Pequeñas
- Una sola responsabilidad
- Pocos parámetros
- Sin efectos secundarios

---

##  Comentarios

- Evitar comentarios innecesarios
- Preferir código claro
- Usar comentarios solo cuando agregan valor

---

##  Formato

- Código ordenado
- Estructura clara
- Buena indentación

---

##  Manejo de errores

- Usar excepciones
- Evitar null

---

##  Testing

- Aplicar TDD
- Tests claros y simples

---

##  Clases

- Pequeñas
- Una sola responsabilidad (SRP)

---

##  Code smells

- Código rígido
- Difícil de modificar
- Repetición innecesaria

---

##  Checklist

- [ ] Funciones pequeñas
- [ ] Nombres claros
- [ ] Sin duplicación
- [ ] Código legible
